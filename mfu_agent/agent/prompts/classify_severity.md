## Задача

На основе предоставленной сервисной документации определи критичность ошибки.

### Уровни критичности

| Уровень    | Описание |
|------------|----------|
| **Critical** | Устройство полностью неработоспособно, требуется немедленное вмешательство сервисного инженера. Примеры: отказ фьюзера, критическая ошибка платы управления, замятие с повреждением механизма. |
| **High**     | Серьёзная деградация функциональности, устройство работает с существенными ограничениями. Примеры: отказ одного из лотков подачи, ошибки сканера, перегрев блока закрепления. |
| **Medium**   | Умеренное влияние на работу, устройство функционирует с незначительными ограничениями. Примеры: ошибки датчиков тонера, предупреждения о ресурсе расходных материалов, периодические замятия. |
| **Low**      | Минимальное влияние, устройство работает штатно. Примеры: информационные сообщения о плановом ТО, предупреждения о скором окончании ресурса. |
| **Info**     | Информационное сообщение без влияния на работоспособность. Примеры: статусные сообщения, уведомления о завершении задания. |

### Правила оценки confidence

- **0.8–1.0** — точное совпадение кода ошибки в документации с описанием критичности
- **0.6–0.8** — код найден, но критичность определена по косвенным признакам
- **0.4–0.6** — похожие ошибки найдены, экстраполяция по аналогии
- **0.2–0.4** — слабое совпадение, низкая уверенность
- **< 0.2** — документация не релевантна, оценка по общим правилам

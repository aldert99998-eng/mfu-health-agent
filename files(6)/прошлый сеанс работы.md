### Этап 5.6 — Trace и память

**1. Trace serialization** (`data_io/models.py`, класс `Trace`):
- `to_json()` → JSON-строка через `model_dump_json(indent=2)`
- `from_json(raw)` → восстановление через `model_validate_json`
- `summary()` → однострочная сводка: mode, session_id, device, HI, tools, llm, tokens, attempts, duration, flagged

**2. MemoryManager** (`agent/memory.py`):
- In-memory хранилище `LearnedPattern` с дедупликацией и мёржем evidence
- `save_pattern(pattern)` → проверка порога evidence, поиск дубликата (scope + observation), мёрж devices
- `get_patterns(scope)` → возвращает паттерны для конкретной модели + fleet-scope
- `to_dict()` / `from_dict()` → для персистенции в Streamlit `session_state`
- Лимит: `max_patterns_per_model` (default 50)

**3. GetLearnedPatternsTool** (`agent/tools/impl.py`):
- 10-й инструмент в реестре (было 9)
- Аргументы: `{model: str | null}`
- Возвращает паттерны из MemoryManager; если memory_manager=None → пустой список

**4. Интеграция в core.py**:
- Конструктор Agent принимает опциональный `memory_manager`
- `run_batch`: после approved reflection → `_save_patterns_to_memory()` сохраняет паттерны из context в MemoryManager
- `_build_messages` (batch): инжектирует паттерны из MemoryManager (по модели устройства) в системный промпт
- `_build_chat_messages` (chat): инжектирует все паттерны из MemoryManager в системный промпт
- Добавлен `_resolve_device_model()` для определения модели из контекста

- Reflection: `temperature=0.1`, `max_tokens=500`

# Тест-план 09 — Нефункциональные требования

**Область:** Performance, Security, Reliability, Observability, Deployment
**Владелец:** QA-инженер NFR + DevOps
**Связанные документы:** `architecture_overview.md` разделы 10, 12

---

## Цель

Проверить, что система:
1. **Укладывается в SLA** по времени и ресурсам.
2. **Безопасна** относительно корпоративных политик.
3. **Устойчива** к частичным сбоям.
4. **Наблюдаема** (логи, метрики, alerts).
5. **Разворачивается** в целевом окружении.

## Критерии приёмки нефункциональных

- Все **P0** пройдены.
- SLA на batch и chat соблюдены (TC-NF-001, 004, 005).
- Утечек памяти нет (TC-NF-015).
- Нет секретов в коде / логах (TC-NF-050..054).
- Docker-деплой отрабатывает на чистой машине (TC-NF-090).
- PDF со шрифтами корректно собирается (TC-NF-092).
- Audit trail tool calls есть (TC-NF-074).
- Healthcheck работает (TC-NF-087).

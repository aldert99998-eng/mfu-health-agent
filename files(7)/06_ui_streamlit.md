# Тест-план 06 — UI Streamlit

**Область:** `app.py`, `pages/`, `state/` — 4 страницы, session_state, singletons
**Владелец:** QA-инженер UI
**Связанные документы:** `architecture_overview.md` раздел 3, 4, `vscode_llm_playbook.md`

---

## Цель

Проверить, что:
1. **Streamlit-приложение стабильно** при типичных действиях пользователя.
2. **Session state** сохраняется между reruns одной сессии.
3. **Singletons через `@st.cache_resource`** — корректно создаются один раз и переживают reruns, переключения, reload.
4. **Все 4 страницы** работают изолированно и согласованно.
5. **Graceful handling** пустых состояний.

## Страницы

1. `1_Dashboard.py` — загрузка, прогресс, просмотр отчёта.
2. `2_Weights.py` — профили, слайдеры.
3. `3_LLM_Chat.py` — чат с агентом + expandable trace.
4. `4_RAG_Admin.py` — управление коллекциями.

## Критерии приёмки UI

- Все **P0** пройдены.
- Singletons через `@st.cache_resource` работают (TC-UI-010..013).
- Переключение вкладок и reload не прерывают работу (TC-UI-023, 024).
- Session_state сохраняется через reruns (TC-UI-060..064).
- Нет XSS (TC-UI-110).
- Graceful handling пустых состояний.

# Тест-план 05 — Трек C: Reporting (HTML + PDF отчёт)

**Область:** `reporting/` — report_builder, pdf_generator, templates, styles
**Владелец:** QA-инженер трека C
**Связанные документы:** `track_C_report_layout.md`, `architecture_overview.md` раздел 6.7

---

## Цель

Проверить, что:
1. **HTML-отчёт собирается полностью** из `HealthResult` + `Trace` + метаданных.
2. **PDF генерируется без искажений** относительно HTML.
3. **Executive summary** (LLM-вызов) корректен и воспроизводим.
4. **Top patterns** (через `find_similar_devices`) группируют проблемы осмысленно.
5. **Шрифты, стили, печать** — работают и внутри Docker.

## Контракты

- **Вход:** `list[HealthResult]`, `Trace` (опц.), `WeightsProfile`, `FleetSummary`.
- **Выход:** `Report` (data model), HTML-строка, PDF-файл.
